HEROKU – QUICK GUIDE

What is Heroku?
Heroku is a cloud Platform as a Service (PaaS) that lets you deploy, run,
and manage applications without managing servers.

Simple idea:
Push your code → Heroku runs it on the internet.

----------------------------------------

What you can deploy on Heroku:
- Python (FastAPI, Flask, Django)
- Node.js
- Java, Ruby, PHP
- PostgreSQL database
- Redis

----------------------------------------

How Heroku works (step by step):
1. Create an app on Heroku
2. Connect GitHub repo or push with Git
3. Heroku detects the language
4. Dependencies are installed automatically
5. App is started
6. You get a public URL

Example URL:
https://your-app-name.herokuapp.com

----------------------------------------

Why developers use Heroku:
- Easy deployment
- No server management
- Environment variables support
- Logs and monitoring
- Built-in PostgreSQL addon

----------------------------------------

Heroku with FastAPI:
Common stack:
- FastAPI
- Uvicorn / Gunicorn
- PostgreSQL
- Alembic migrations

Good for:
- APIs
- Job boards
- Real estate apps
- Portfolio projects

----------------------------------------

Downsides:
- Free tier removed
- Paid plans only
- Not ideal for very large systems
- Cheap plans may sleep when inactive

----------------------------------------

Alternatives to Heroku:
- Railway
- Render
- Fly.io
- DigitalOcean App Platform

----------------------------------------

When to use Heroku:
- Learning backend development
- Portfolio & demo projects
- Small to medium APIs
- Fast deployment needs

----------------------------------------

One-line summary:
Heroku is a cloud platform that makes deploying backend applications easy
without managing servers.
